## 3. 测试环境

### 3.1 构建配置

```bash
# 配置
mkdir build && cd build
cmake .. -DPET_CORE_BUILD_TESTS=ON -DBUILD_TESTS=ON

# 构建
cmake --build . --config Release

# 运行测试
ctest --output-on-failure
# 或
./run_tests.ps1
```

### 3.2 依赖项

- CMake >= 3.16
- C++17 兼容编译器 (MSVC/GCC/Clang)
- Google Test v1.14.0 (自动下载)
- Qt5.15+ (UI 测试)

## 4. 测试执行策略

### 4.1 持续集成

- 每次提交自动运行单元测试
- PR 合并前必须通过所有测试
- 覆盖率低于 80% 的 PR 需要说明理由

### 4.2 测试运行频率

| 测试类型 | 频率 | 触发条件 |
|----------|------|----------|
| 单元测试 | 每次提交 | git push |
| 集成测试 | 每次提交 | git push |
| UI 测试 | 每日构建 | 定时任务 |
| 性能测试 | 每周 | 定时任务 |

### 4.3 测试报告

- 位置：`build/tests/test_report_*.md`
- 格式：Markdown
- 内容：测试结果、统计、覆盖率

## 5. 缺陷管理

### 5.1 缺陷优先级

| 优先级 | 描述 | 响应时间 |
|--------|------|----------|
| P0 | 阻塞性缺陷，无法继续开发 | 立即 |
| P1 | 严重缺陷，核心功能失效 | 24 小时 |
| P2 | 一般缺陷，部分功能异常 | 3 天 |
| P3 | 轻微缺陷，不影响使用 | 下个迭代 |
