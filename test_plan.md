# 电子宠物项目 - 测试计划

## 1. 测试目标

确保电子宠物应用的核心功能稳定可靠，包括宠物成长、积分系统、兑换系统和数据持久化。

---

## 2. 功能测试范围

| 模块 | 测试项 | 优先级 |
|------|--------|--------|
| **宠物成长** | 成长逻辑、状态变化、年龄计算 | P0 |
| **积分系统** | 积分获取、累计、显示 | P0 |
| **兑换系统** | 道具兑换、积分扣除、库存管理 | P0 |
| **数据持久化** | 本地存储、重启恢复、数据一致性 | P0 |
| **UI 交互** | 界面响应、动画效果、异常提示 | P1 |

---

## 3. 测试环境要求

### 3.1 硬件环境
- **最低配置**: 
  - CPU: 双核 2.0GHz
  - 内存：4GB RAM
  - 存储：500MB 可用空间
- **推荐配置**:
  - CPU: 四核 2.5GHz+
  - 内存：8GB RAM
  - 存储：1GB 可用空间

### 3.2 软件环境
- **操作系统**: Windows 10/11, Linux (Ubuntu 20.04+)
- **运行环境**: 
  - Qt5.15+ 运行时库
  - C++ 运行时库 (MSVC 2019+)
- **测试工具**:
  - Qt Test Framework
  - 手动测试检查表

### 3.3 测试数据
- 预设宠物初始状态数据
- 边界值测试数据（负数、溢出、空值）
- 正常流程测试数据

## 5. 测试策略

### 5.1 测试类型
- **单元测试**: 核心逻辑函数（积分计算、成长算法）
- **集成测试**: 模块间接口（积分→兑换→库存）
- **系统测试**: 完整流程（启动→互动→保存→重启）
- **边界测试**: 极端值、异常输入

### 5.2 测试方法
- **黑盒测试**: 基于需求规格验证功能
- **白盒测试**: 代码覆盖率检查（目标 >80%）
- **探索性测试**: 非预期操作路径

### 5.3 通过标准
- 所有 P0 测试用例通过率 100%
- P1 测试用例通过率 >95%
- 无崩溃性 Bug
- 数据持久化验证通过

## 6. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 数据丢失 | 高 | 增加备份机制测试 |
| 积分计算错误 | 高 | 边界值全覆盖测试 |
| UI 崩溃 | 中 | 异常输入防护测试 |
| 性能问题 | 低 | 长时间运行监控 |
